Interactive Baseball Simulator

Simulates the play of baseball games using a statistical probability model, data mining, and artifical intelligence to make informed dynamic game time decisions

Download this project as a .zip file Download this project as a tar.gz file

Goals

    The goals of this project are primarily to give the team members a significant amount of experience with system design and implementation that can be used in real world situations, and secondarily to provide a meaningful quality system that can be used in the real world.

  • Develop a product to simulate a single baseball game, an entire regular season, and the playoffs
  • Greatly enhance and expand the knowledge of the development team
  • Provide an interface to allow web applications, mobile apps, and 3rd parties to interact with the simulator

Description

Simulation

The completed system will simulate at bats based on a statistical probability model. The following statistics will be used to calculate the results. The exact forumlas have yet to be developed, but I am confident that this will be a minor task.

  • The number of hits allowed, walks, and strikeouts for a pitcher
  • The propertion of hits, doubles, triples, and home runs to outs for a batter

Dynamic Decisions

The final version of the system will also include components for data mining and artificial intelligence to make the simulator more dynamic. In the final version, the system will support the following features:

  • The ability to decide when a team should go to the bullpen
  • The ability to determine when to use a pinch hitter based on the current situation
  • The ability to calculate effective pitching and batting stats by analyzing recent and historical performances

Limitations and Assumptions

This is is a simulator, it can never be precise enough to model human behavoir, but nonethless it can point you in the right direction. The following are major assumptions this project is based on:

  • The simulator is governed by chance, therefore there are no biased decisions one way or another so the system accurately describes the probabile outcome
  • Long term statistics of a player are useful, but using short term statistics is more realistic
  • The common trends of hitters and pitchers over time are useful in calculating effective batting and pitching probabilities

Schedule

Scheduling is the major constraint for this project. The project will be worked on for short bursts of three months and sit idle for 4 to 6 months before being worked on again. The work will be broken down into 3 major versions:

  1. Develop a basic simulator system, Due: 8/24/2012
  2. Develop a web application that can interact with the simulator, Due 1/18/2013
  3. Develop a mobile application that can interact with the simulator, Due 6/7/2013

Development Platforms and Technologies

The system will use some servers that run on a variant of the UNIX operating system and others that run Microsoft. The following shows what combination of platforms, programming languages, and technologies will be used for which components:

  • A Server running on UNIX will support the main simulator component written in C
  • A server running on Microsoft will host a web application built using ASP.NET Model View Controller and Windows Communication Foundation
  • The mobile application will be built for Android OS Version 2.3.3
